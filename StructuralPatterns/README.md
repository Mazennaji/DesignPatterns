# Structural Design Patterns (Java)

## ğŸ“Œ Overview

**Structural Design Patterns** focus on how **classes and objects are composed** to form **larger, flexible, and efficient structures**.
Rather than concentrating on object creation or behavior, these patterns emphasize **relationships, composition, and structure**.

Structural patterns help ensure that changes in one part of the system **do not ripple through the entire codebase**, making systems **robust, scalable, and easy to maintain**.

---

## ğŸ¯ Why Structural Patterns?

Structural patterns are used when:

* You want to **simplify complex object structures**
* You need to **adapt incompatible interfaces**
* You want to **add functionality without modifying existing code**
* You need to **decouple abstractions from implementations**
* You want to **optimize memory usage** for large numbers of objects

They improve:

* Flexibility
* Maintainability
* Reusability
* System scalability
* Code readability

## ğŸ§  Core Principles Used

Structural patterns strongly rely on:

* **Composition over inheritance**

* **Single Responsibility Principle (SRP)**
* **Open/Closed Principle (OCP)**
* **Dependency Inversion Principle (DIP)**

## ğŸ§© Common Structural Design Patterns

| Pattern Name  | Purpose                                               |
| ------------- | ----------------------------------------------------- |
| **Adapter**   | Convert one interface into another                    |
| **Bridge**    | Decouple abstraction from implementation              |
| **Composite** | Treat individual objects and compositions uniformly   |
| **Decorator** | Add behavior dynamically                              |
| **Facade**    | Provide a simplified interface to a complex subsystem |
| **Flyweight** | Reduce memory usage by sharing objects                |
| **Proxy**     | Control access to another object                      |
